PART 1 — SPLUNK: CREATE GPS DATA STORE + HEC TOKEN
================================================================================

1a. Create a dedicated Splunk index for PTA data.

    Via Splunk Web  (http://brannigan.taila277ca.ts.net:8000):
      Settings → Indexes → New Index
        Index Name:   pta
        Index Type:   Events
        Max Size:     leave at default (500 MB fine for POC)
      Save.

    Via REST (run on brannigan terminal):
      curl -k -u admin:$SPLUNK_PASSWORD \
        -X POST https://127.0.0.1:8089/services/data/indexes \
        -d "name=pta" -d "datatype=event"

1b. Create a dedicated HEC token for the GPS feed.

    Via Splunk Web:
      Settings → Data Inputs → HTTP Event Collector → New Token
        Name:         pta_gps_feed
        Source type:  (create new) → pta:gps_location
        Index:        pta
      Click through to the end — COPY the generated token value.

    Via REST:
      curl -k -u admin:$SPLUNK_PASSWORD \
        -X POST https://127.0.0.1:8089/services/data/inputs/http \
        -d "name=pta_gps_feed" \
        -d "sourcetype=pta:gps_location" \
        -d "index=pta"

      The token value is in the response JSON under "token".

    ── YOUR VALUES ────────────────────────────────────────────────────────────
    SPLUNK_HEC_URL   =  https://brannigan.taila277ca.ts.net:8088/services/collector/event
    SPLUNK_HEC_TOKEN =  REDACTED_SPLUNK_HEC_TOKEN
    ───────────────────────────────────────────────────────────────────────────

1c. Enable CORS on the Splunk HEC so the mobile browser can POST cross-origin.

    Via Splunk Web:
      Settings → Data Inputs → HTTP Event Collector → Global Settings
        ☑ Enable SSL
        ☑ Enable Cross-Origin Resource Sharing (CORS)
        CORS origin:  *   (for POC; restrict to your AI Studio URL in production)
      Save.

    Via inputs.conf on brannigan (alternative):
      sudo -u splunk bash -c "cat >> /opt/splunk/etc/system/local/inputs.conf <<'EOF'

      [http]
      enableSSL = 1
      crossOriginPosting = 1
      allowedOrigins = *
      EOF"
      sudo systemctl restart Splunkd

1d. Verify the HEC token works from brannigan terminal:

      curl -k https://127.0.0.1:8088/services/collector/event \
        -H "Authorization: Splunk <YOUR_TOKEN>" \
        -H "Content-Type: application/json" \
        -d '{
          "sourcetype": "pta:gps_location",
          "index": "pta",
          "event": {
            "device_id": "test-device",
            "lat": -31.9505,
            "lon": 115.8605,
            "accuracy": 10.0,
            "altitude_m": 45.0,
            "speed_kmh": 0,
            "ts": "2026-06-22T10:00:00Z"
          }
        }'

    Expected response:  {"text":"Success","code":0}

1e. Confirm the test event landed in Splunk:

    In Splunk Search:
      index=pta sourcetype="pta:gps_location"
      | table _time, device_id, lat, lon, accuracy

    You should see the test row.

PART 2 — GOOGLE MAPS PLATFORM API KEY
================================================================================

2a. Open Google Cloud Console:
      https://console.cloud.google.com

2b. Create a new project (or use an existing one):
      Top bar → project selector → NEW PROJECT
        Project name:  PTA Splunk Dash
      Click Create.

2c. Enable the Maps JavaScript API:
      Left menu → APIs & Services → Library
      Search: "Maps JavaScript API"
      Click it → Enable.

2d. Create an API key:
      Left menu → APIs & Services → Credentials
      + CREATE CREDENTIALS → API key
      A key is generated immediately — copy it.

2e. (Recommended) Restrict the key to prevent misuse:
      Click the key name → edit
      Application restrictions → HTTP referrers (websites)
      Add:
        https://brannigan.taila277ca.ts.net:8000/*   (Splunk Web)
        http://brannigan.taila277ca.ts.net:5000/*    (ICP Agent web app)
        https://*.aistudio.google.com/*              (AI Studio preview)
      API restrictions → Restrict key → Maps JavaScript API only.
      Save.

    ── YOUR VALUE ─────────────────────────────────────────────────────────────
    GOOGLE_MAPS_PLATFORM_KEY  =  <key from step 2d>
    ───────────────────────────────────────────────────────────────────────────

PART 6 — MOBILE PWA: GPS SEND CODE
================================================================================
Add the following JavaScript to your Google AI Studio PWA.
This is the "Send Location" button logic.

---------- gps_sender.js (add to your PWA) ----------

const HEC_URL   = process.env.SPLUNK_HEC_URL;
const HEC_TOKEN = process.env.SPLUNK_HEC_TOKEN;

// Call this function when the user taps "Send Location"
async function sendLocationToSplunk() {
  const statusEl = document.getElementById('gps-status');  // optional UI element

  if (!navigator.geolocation) {
    alert('Geolocation is not supported by this browser.');
    return;
  }

  statusEl && (statusEl.textContent = 'Getting GPS fix...');

  navigator.geolocation.getCurrentPosition(
    async (position) => {
      const { latitude, longitude, accuracy, altitude, speed, heading } = position.coords;

      const payload = {
        sourcetype: 'pta:gps_location',
        index:      'pta',
        event: {
          device_id:   getDeviceId(),           // see helper below
          lat:         latitude,
          lon:         longitude,
          accuracy:    accuracy,                 // metres
          altitude_m:  altitude   ?? null,
          speed_kmh:   speed != null ? (speed * 3.6).toFixed(1) : null,
          heading:     heading     ?? null,
          ts:          new Date().toISOString(),
        }
      };

      try {
        const res = await fetch(HEC_URL, {
          method:  'POST',
          headers: {
            'Authorization': 'Splunk ' + HEC_TOKEN,
            'Content-Type':  'application/json',
          },
          body: JSON.stringify(payload),
        });

        const data = await res.json();
        if (data.code === 0) {
          statusEl && (statusEl.textContent =
            '✓ Sent  ' + latitude.toFixed(5) + ', ' + longitude.toFixed(5) +
            '  (±' + accuracy.toFixed(0) + 'm)  @ ' + new Date().toLocaleTimeString());
        } else {
          statusEl && (statusEl.textContent = 'Splunk error: ' + data.text);
        }
      } catch (err) {
        statusEl && (statusEl.textContent = 'Network error: ' + err.message);
        console.error('HEC POST failed', err);
      }
    },
    (error) => {
      const msgs = {
        1: 'Location permission denied — allow in browser settings.',
        2: 'Position unavailable — try moving outdoors.',
        3: 'GPS timed out — try again.',
      };
      statusEl && (statusEl.textContent = msgs[error.code] || 'GPS error ' + error.code);
    },
    {
      enableHighAccuracy: true,
      timeout:            10000,   // 10 second GPS fix deadline
      maximumAge:         0,       // always get a fresh fix
    }
  );
}

// Stable per-device ID stored in localStorage
function getDeviceId() {
  let id = localStorage.getItem('pta_device_id');
  if (!id) {
    id = 'pta-' + Math.random().toString(36).slice(2, 8);
    localStorage.setItem('pta_device_id', id);
  }
  return id;
}

---------- minimal HTML button to add to your PWA ----------

<button onclick="sendLocationToSplunk()" style="
  padding: 16px 32px; font-size: 18px; font-weight: bold;
  background: #1a73e8; color: white; border: none; border-radius: 8px;
  cursor: pointer; width: 100%;">
  📍 Send My Location to Splunk
</button>
<p id="gps-status" style="font-family: monospace; font-size: 13px; color: #555;"></p>

---------- end of GPS code ----------

================================================================================
TROUBLESHOOTING
================================================================================

Problem: Browser shows CORS error on fetch to HEC
  Fix:    Repeat step 1c — enable crossOriginPosting in Splunk HEC global settings.
          Check browser DevTools Network tab for the preflight OPTIONS response.

Problem: fetch returns "Failed to fetch" or net::ERR_CERT_AUTHORITY_INVALID
  Fix:    The phone browser is rejecting Splunk's self-signed cert.
          On the phone, visit https://brannigan.taila277ca.ts.net:8088 directly
          and accept the cert exception. Then retry the PWA.
          Long-term: install a proper TLS cert (Let's Encrypt via Tailscale HTTPS).

Problem: GPS status shows "Location permission denied"
  Fix:    On the phone: Settings → Browser → Location → Allow for this site.
          PWAs require HTTPS to use geolocation — ensure the AI Studio preview
          URL is served over HTTPS.

Problem: Map shows "For development purposes only" watermark
  Fix:    Billing is not enabled on the Google Cloud project.
          Console → Billing → link a billing account.  Maps API is free under
          ~28,000 loads/month — well within POC limits.

Problem: Map stays on Perth default, no marker
  Fix:    Open browser DevTools on the Splunk Web page (F12).
          Check Console for JS errors.  Run the Splunk search manually:
            index=pta sourcetype="pta:gps_location" | stats latest(lat) AS lat ...
          If no results, GPS pings haven't landed yet.

Problem: Dashboard "source not found" after XML import
  Fix:    Check the view name: it must match the filename without extension.
          In Splunk Web go to Settings → User Interface → Dashboards and confirm
          "PTA GPS Tracker" appears in the list.

Problem: AI Studio env vars not available at runtime
  Fix:    Some AI Studio builds require env vars to be prefixed VITE_ or NEXT_PUBLIC_
          to be exposed to browser code.  Check the AI Studio docs for your project
          type, then prefix accordingly (e.g. VITE_SPLUNK_HEC_TOKEN).


================================================================================
SECURITY NOTES
================================================================================
  - The HEC token grants write-only access to Splunk (cannot read or admin).
    Safe to embed in a PWA for POC; for production use a backend proxy.
  - Restrict the Google Maps API key to your specific HTTP referrers (step 2e)
    to prevent quota abuse.
  - Enable billing alerts in Google Cloud Console so you get notified if
    the key is ever misused.
  - For production: set CORS origin to the specific AI Studio PWA URL rather
    than wildcard (*).

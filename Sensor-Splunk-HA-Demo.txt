        -d sourcetype=icp:ext_sensor \

        -d '{"sourcetype":"icp:ext_sensor","event":{"sensor":"test","value":42}}'

PART 3 — COLLEAGUE: SENSOR FEED SCRIPT
================================================================================
Run this on the colleague's device (Python 3.8+).

3a. Install dependencies:
      pip install requests

3b. Create the file  sensor_feed.py  with the following content:

---------- sensor_feed.py ----------
import time, json, random, requests, datetime, warnings
warnings.filterwarnings("ignore")   # suppress SSL verify warnings

SPLUNK_HEC_URL   = "https://brannigan.taila277ca.ts.net:8088/services/collector/event"
SPLUNK_HEC_TOKEN = "<TOKEN-FROM-STEP-2b>"
SENSOR_ID        = "colleague-env-01"      # change to identify your device
LOCATION         = "Remote Office"         # label shown in dashboards
POLL_SECONDS     = 30                      # how often to push readings

def read_sensor():
    """Replace with real sensor readings (GPIO, i2c, MQTT, etc.)."""
    return {
        "temperature_c":  round(random.uniform(18.0, 26.0), 1),
        "humidity_pct":   round(random.uniform(40.0, 65.0), 1),
        "co2_ppm":        random.randint(400, 1200),
        "occupancy":      random.choice([0, 1]),
    }

def push_to_splunk(data):
    payload = {
        "time":       time.time(),
        "sourcetype": "icp:ext_sensor",
        "source":     SENSOR_ID,
        "host":       SENSOR_ID,
        "event": {
            "sensor_id":  SENSOR_ID,
            "location":   LOCATION,
            "ts":         datetime.datetime.utcnow().isoformat() + "Z",
            **data
        }
    }
    r = requests.post(
        SPLUNK_HEC_URL,
        headers={"Authorization": f"Splunk {SPLUNK_HEC_TOKEN}"},
        data=json.dumps(payload),
        verify=False,
        timeout=10,
    )
    print(f"[{payload['event']['ts']}] pushed: {data}  → HTTP {r.status_code}")

if __name__ == "__main__":
    print(f"Starting sensor feed from {SENSOR_ID} → Splunk every {POLL_SECONDS}s")
    while True:
        try:
            push_to_splunk(read_sensor())
        except Exception as e:
            print(f"ERROR: {e}")
        time.sleep(POLL_SECONDS)
---------- end of sensor_feed.py ----------

3c. Run it:
      python sensor_feed.py

    You should see timestamped lines confirming HTTP 200 responses from Splunk.

3d. (Optional) Run as a background service on Linux:
      nohup python sensor_feed.py >> sensor_feed.log 2>&1 &

PART 4 — SPLUNK: VERIFY + CREATE SAVED SEARCH
================================================================================
Do this on Stanley's server.

4a. Confirm events are arriving. In Splunk Search:

      index=main sourcetype="icp:ext_sensor"
      | head 20
      | table _time, sensor_id, location, temperature_c, humidity_pct, co2_ppm, occupancy

    You should see rows from colleague-env-01.

4b. Create a saved search for "latest ext sensor reading" so Home Assistant
    can poll it via REST:

      Name:   ICP_ExtSensor_Latest
      Search:
        index=main sourcetype="icp:ext_sensor"
        | stats latest(temperature_c) AS temperature_c
                latest(humidity_pct)  AS humidity_pct
                latest(co2_ppm)       AS co2_ppm
                latest(occupancy)     AS occupancy
                latest(_time)         AS last_seen
          BY sensor_id, location
      Schedule: every 1 minute

    Via Splunk Web: Search → Save As → Saved Search → set schedule.
    Via REST API:
      curl -k -u admin:$SPLUNK_PASSWORD \
        -X POST https://127.0.0.1:8089/services/saved/searches \
        -d "name=ICP_ExtSensor_Latest" \
        -d "search=index=main sourcetype=\"icp:ext_sensor\" | stats latest(temperature_c) AS temperature_c latest(humidity_pct) AS humidity_pct latest(co2_ppm) AS co2_ppm latest(occupancy) AS occupancy latest(_time) AS last_seen BY sensor_id, location" \
        -d "cron_schedule=* * * * *" \
        -d "is_scheduled=1" \
        -d "enableSched=1"

4c. Test the saved search result via the Splunk REST API (returns JSON):

      curl -k -u admin:$SPLUNK_PASSWORD \
        "https://127.0.0.1:8089/services/search/jobs/export?search=savedsearch+ICP_ExtSensor_Latest&output_mode=json"

    Alternatively run an ad-hoc search via REST that HA will use:

      curl -k -u admin:$SPLUNK_PASSWORD \
        -X POST https://127.0.0.1:8089/services/search/jobs \
        -d "search=search index=main sourcetype=\"icp:ext_sensor\" | stats latest(temperature_c) AS temp, latest(humidity_pct) AS hum, latest(co2_ppm) AS co2 BY sensor_id" \
        -d "output_mode=json" \
        -d "exec_mode=oneshot"

PART 5 — HOME ASSISTANT: REST SENSOR SETUP
================================================================================
Home Assistant is on the colleague's device (or separate RPi) on the same
Tailnet.  We use HA's REST sensor integration to pull data from Splunk.

NOTE: Splunk's management API (8089) requires HTTP Basic auth and returns XML
by default.  We use output_mode=json and exec_mode=oneshot for a single-shot
search job — Home Assistant's REST sensor handles this cleanly.

5a. In Home Assistant, go to:
      Settings → Devices & Services → Helpers
    OR edit  configuration.yaml  directly.

5b. Add a REST sensor block to  configuration.yaml:

---------- configuration.yaml snippet ----------
rest:
  - resource: "https://brannigan.taila277ca.ts.net:8089/services/search/jobs/export"
    method: POST
    payload: >
      search=search index%3Dmain sourcetype%3D"icp%3Aext_sensor"
      %7C stats latest(temperature_c) AS temp latest(humidity_pct) AS hum
      latest(co2_ppm) AS co2 latest(occupancy) AS occ BY sensor_id
      &output_mode=json&exec_mode=oneshot
    headers:
      Content-Type: application/x-www-form-urlencoded
      Authorization: !secret splunk_basic_auth
    verify_ssl: false
    scan_interval: 60
    sensor:
      - name: "ICP Ext Sensor Temperature"
        value_template: "{{ value_json.results[0].temp }}"
        unit_of_measurement: "°C"
        device_class: temperature
      - name: "ICP Ext Sensor Humidity"
        value_template: "{{ value_json.results[0].hum }}"
        unit_of_measurement: "%"
        device_class: humidity
      - name: "ICP Ext Sensor CO2"
        value_template: "{{ value_json.results[0].co2 }}"
        unit_of_measurement: "ppm"
        device_class: carbon_dioxide
      - name: "ICP Ext Sensor Occupancy"
        value_template: "{{ value_json.results[0].occ }}"
---------- end of snippet ----------

5c. Add the Splunk credential to  secrets.yaml  (base64 of "admin:password"):

      # In a terminal on the HA host:
      echo -n "admin:<SPLUNK_PASSWORD>" | base64
      # outputs e.g. YWRtaW46cGFzc3dvcmQ=

    In secrets.yaml:
      splunk_basic_auth: "Basic YWRtaW46cGFzc3dvcmQ="

    NOTE: If your HA version supports it, use the Splunk integration from
    HACS instead — it wraps these REST calls in a friendlier UI.  The manual
    REST approach above works on any HA version.

5d. Restart Home Assistant:
      Settings → System → Restart

5e. Verify sensors appear:
      Developer Tools → States → search "ICP Ext Sensor"
    You should see Temperature, Humidity, CO2, and Occupancy with real values.

5f. Add sensors to a HA Dashboard:
      Settings → Dashboards → Edit → Add Card → Entities
        - ICP Ext Sensor Temperature
        - ICP Ext Sensor Humidity
        - ICP Ext Sensor CO2
        - ICP Ext Sensor Occupancy

    For a richer display use "Glance" or "Gauge" card types.

================================================================================
TROUBLESHOOTING
================================================================================

Problem: curl to :8088 times out from colleague's device
  Fix:    On brannigan — sudo ufw allow in on tailscale0 to any port 8088
          Confirm with: sudo ufw status | grep 8088

Problem: HEC returns 403 Forbidden
  Fix:    Wrong token.  Re-check Settings → Data Inputs → HTTP Event Collector
          and ensure the token is Enabled.

Problem: HEC returns 400 — incorrect data format
  Fix:    The payload must be valid JSON with an "event" key.
          Check sensor_feed.py for syntax errors.

Problem: Splunk REST search returns XML instead of JSON
  Fix:    Ensure "&output_mode=json" is in the URL/payload.

Problem: HA sensor shows "unknown" after restart
  Fix:    Check HA logs:  Settings → System → Logs → search "rest"
          Common issues: SSL verify failure (set verify_ssl: false),
          base64 auth wrong, Splunk search returning 0 results.

Problem: HA can't reach brannigan.taila277ca.ts.net:8089
  Fix:    Ensure HA host is also on the Tailnet.  Test with:
            curl -k -u admin:PASSWORD https://brannigan.taila277ca.ts.net:8089/services/server/info


================================================================================
SECURITY NOTES
================================================================================
  - The HEC token grants write-only access to Splunk — safe to share with
    the colleague; it cannot be used to read or administer Splunk.
  - The Splunk management port (8089) requires admin credentials and is
    only needed by Home Assistant for read queries.  Restrict to Tailscale
    IPs only (ufw rule above) so it is never exposed to the public internet.
  - Never commit tokens or passwords to the GitHub repo.  Use .env / secrets.yaml.
  - For production, create a Splunk role with read-only search permissions
    and use that account instead of admin in HA's secrets.yaml.

CONCEPT
-------
A Google AI Studio Progressive Web App (PWA) installed on a mobile phone
captures live GPS coordinates and transmits them to Splunk Enterprise via a
secure FastAPI proxy running on Brannigan. A Google Maps dashboard — served
by the same FastAPI server and embedded in a Splunk dashboard — shows the
phone's real-time position updated every 15 seconds.

Key design decisions:
  • The phone and server share a Tailscale private network — no public
    internet exposure of Splunk or internal APIs.
  • FastAPI acts as a secure proxy: the phone sends plain JSON, the server
    adds the Splunk HEC token and forwards. The token never touches the client.
  • FastAPI serves the Google Maps page directly, avoiding Splunk 10.x CSP
    restrictions that block inline JavaScript in SimpleXML HTML panels.
  • Tailscale provides a valid TLS certificate for the server hostname,
    resolving mixed-content browser blocks (AI Studio preview is HTTPS,
    so all fetch() calls must also be HTTPS).

COMPONENT DETAILS
================================================================================

── 1. MOBILE APP  (Google AI Studio PWA) ──────────────────────────────────────

  File:        app.tsx
  Framework:   React + TypeScript
  GPS method:  navigator.geolocation.watchPosition()
                 enableHighAccuracy: true
                 maximumAge: 5000ms
                 timeout: 10000ms

  Vehicle ID:  Generated once as VHL-XXXX, stored in localStorage.
               User can change it in Settings (not while tracking).

  Payload sent to server:
    {
      latitude:  number,      // decimal degrees
      longitude: number,      // decimal degrees
      accuracy:  number,      // metres (radius of 68% confidence)
      speed:     number|null, // metres per second (from Geolocation API)
      heading:   number|null, // degrees clockwise from true north
      timestamp: number,      // Date.now() — milliseconds epoch
      vehicleId: string       // e.g. "VHL-0042" or "STAN-PHONE"
    }

  Endpoint:  https://brannigan.taila277ca.ts.net:5000/api/pta/telemetry
  Method:    POST, Content-Type: application/json

  CRITICAL — Why full HTTPS URL, not relative /api/telemetry:
    AI Studio preview is served from https://aistudio.google.com.
    A relative URL (/api/telemetry) resolves to Google's servers, which
    cannot reach the Tailscale-only Brannigan.
    Using an absolute URL forces the browser to POST directly to Brannigan
    over the Tailscale VPN from the phone.

  CRITICAL — Why HTTPS not HTTP:
    Browsers enforce mixed-content policy: an HTTPS page cannot make
    HTTP fetch() calls. AI Studio preview is HTTPS, so the target URL
    must also be HTTPS. Tailscale cert resolves this without self-signed
    certificate warnings.

  transmitToSplunk function (key change from original):
    // Original (broken — goes to Google Cloud, cannot reach Tailscale):
    fetch('/api/telemetry', { ... })

    // Fixed (direct to Brannigan via Tailscale, HTTPS):
    fetch('https://brannigan.taila277ca.ts.net:5000/api/pta/telemetry', { ... })

================================================================================
LESSONS LEARNED / GOTCHAS
================================================================================

1. MIXED CONTENT BLOCK (most important)
   AI Studio preview (HTTPS) cannot fetch HTTP endpoints.
   Browser silently drops the request — no error shown to user,
   Packets Sent counter stays at 0, nothing reaches the server.
   Fix: HTTPS everywhere. Use `tailscale cert` for a valid cert.

2. RELATIVE URL GOES TO GOOGLE'S CLOUD
   fetch('/api/telemetry') in the AI Studio app resolves to
   https://aistudio.google.com/api/telemetry — Google's servers,
   which cannot reach Tailscale-only addresses.
   Fix: Always use the full absolute URL.

3. SPLUNK HTML PANEL CSP (Splunk 10.x)
   SimpleXML encoded="0" HTML panels do not execute inline JavaScript
   in Splunk 10.x due to Content Security Policy restrictions.
   The raw HTML/JS is rendered as escaped text.
   Fix: Serve the map from FastAPI and embed via encoded="1" iframe.

4. SPLUNK WEB PORT IS 8008 (not 8000)
   This installation runs Splunk Web on port 8008.
   Always confirm with: sudo ss -tlnp | grep splunk

5. ANDROID BLOCKS PING (ICMP)
   `ping` in Termux/Android always fails for non-root — not a network issue.
   Use curl to test connectivity instead.

6. WRONG HEC TOKEN
   The existing .env had SPLUNK_HEC_TOKEN pointing to the meraki-switch
   HEC input. A dedicated pta_gps_feed token was created and hardcoded
   as _PTA_HEC_TOKEN in the FastAPI code to keep the data sources clean.

7. DASHBOARD SHARING
   Dashboards created via REST API default to user-level sharing (private).
   Must explicitly set sharing=app via the ACL endpoint after creation.

8. head 1 WITHOUT sort
   `stats ... BY vehicleId | head 1` returns an arbitrary row.
   Always add `| sort -ts` before `| head 1` to get the most recent vehicle.

9. JSON FIELD EXTRACTION
   Splunk auto-extracts JSON fields from _raw for this sourcetype.
   Adding `| spath` on top causes duplicate multi-value fields.
   Do not use spath — the fields are already available.

10. TAILSCALE IP vs HOSTNAME
    The Tailscale cert is issued for the hostname (brannigan.taila277ca.ts.net),
    not the IP (100.81.70.37). Using the IP with HTTPS causes cert mismatch.
    Always use the full hostname in URLs.

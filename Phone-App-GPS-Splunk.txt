CONCEPT
-------
A Google AI Studio Progressive Web App (PWA) installed on a mobile phone
captures live GPS coordinates and transmits them to Splunk Enterprise via a
secure FastAPI proxy running on Brannigan. A Google Maps dashboard — served
by the same FastAPI server — shows the real-time position, full route trace,
timestamped breadcrumb markers, and incident flags updated every 15 seconds.

Key design decisions:
  • The phone and server share a Tailscale private network — no public
    internet exposure of Splunk or internal APIs.
  • FastAPI acts as a secure proxy: the phone sends plain JSON, the server
    adds the Splunk HEC token and forwards. The token never touches the client.
  • FastAPI serves the Google Maps page directly, avoiding Splunk 10.x CSP
    restrictions that block inline JavaScript in SimpleXML HTML panels.
  • Tailscale provides a valid TLS certificate for the server hostname,
    resolving mixed-content browser blocks (AI Studio preview is HTTPS,
    so all fetch() calls must also be HTTPS).
  • Incidents are flagged on the phone and sent as part of the normal GPS
    telemetry payload — no separate server endpoint or data path required.

COMPONENT DETAILS

── 1. MOBILE APP  (Google AI Studio PWA) ──────────────────────────────────────

  File:        app.tsx
  Framework:   React + TypeScript
  GPS method:  navigator.geolocation.watchPosition()
                 enableHighAccuracy: true
                 maximumAge: 5000ms
                 timeout: 10000ms

  Vehicle ID:  Generated once as VHL-XXXX, stored in localStorage.
               User can change it in Settings (not while tracking).

  Standard payload sent to server:
    {
      latitude:  number,      // decimal degrees
      longitude: number,      // decimal degrees
      accuracy:  number,      // metres (radius of 68% confidence)
      speed:     number|null, // metres per second (from Geolocation API)
      heading:   number|null, // degrees clockwise from true north
      timestamp: number,      // Date.now() — milliseconds epoch
      vehicleId: string       // e.g. "VHL-0042" or "STAN-PHONE"
    }

  Incident payload (additional fields, only present when flagged):
    {
      ...standard fields...,
      incident:     true,     // boolean — omitted on normal pings
      incidentNote: string    // optional free-text description
    }

  Incident flag flow in app.tsx:
    • State: isIncident (boolean, default false), incidentNote (string)
    • "Mark Incident" button sets isIncident = true
    • On next GPS send: incident + incidentNote are spread into payload
    • Flag auto-clears after send so subsequent pings are normal again
    • Spread pattern: ...(isIncident && { incident: true })
                      ...(isIncident && incidentNote && { incidentNote: ... })

  Endpoint:  https://brannigan.taila277ca.ts.net:5000/api/pta/telemetry
  Method:    POST, Content-Type: application/json

  CRITICAL — Why full HTTPS URL, not relative /api/telemetry:
    AI Studio preview is served from https://aistudio.google.com.
    A relative URL (/api/telemetry) resolves to Google's servers, which
    cannot reach the Tailscale-only Brannigan.
    Using an absolute URL forces the browser to POST directly to Brannigan
    over the Tailscale VPN from the phone.

  CRITICAL — Why HTTPS not HTTP:
    Browsers enforce mixed-content policy: an HTTPS page cannot make
    HTTP fetch() calls. AI Studio preview is HTTPS, so the target URL
    must also be HTTPS. Tailscale cert resolves this without self-signed
    certificate warnings.

── 3. FASTAPI  (icp_observability_agent.py) ───────────────────────────────────

  Service:     icp-agent.service  (systemd)
  Port:        5000  (HTTPS)
  SSL cert:    /etc/ssl/icp/brannigan.taila277ca.ts.net.crt

  CORS:        CORSMiddleware — allow_origins=["*"]

  ── POST /api/pta/telemetry ──────────────────────────────────────────────────
  Receives phone GPS payload (including optional incident fields), wraps in
  Splunk HEC envelope, forwards unchanged.
  HEC token: _PTA_HEC_TOKEN (env: PTA_HEC_TOKEN)
  Splunk envelope:
    { "sourcetype": "pta:gps_location", "index": "pta", "event": <payload> }
  Returns: { "status": "ok", "splunk": { "text": "Success", "code": 0 } }

  ── GET /api/pta/gps ─────────────────────────────────────────────────────────
  Returns the most recent GPS fix for any vehicle across ALL time
  (not limited to recent window — shows last known position even if idle).
  SPL:
    search index=pta sourcetype="pta:gps_location"
    | stats latest(latitude) AS lat latest(longitude) AS lon
            latest(accuracy) AS accuracy latest(speed) AS speed_ms
            latest(heading) AS heading latest(_time) AS ts BY vehicleId
    | sort -ts | head 1
  earliest_time: 0 (all time)
  Returns: { found, device_id, lat, lon, accuracy, speed_kmh, heading, ts }
  Note: speed converted m/s → km/h (×3.6)

  ── GET /api/pta/gps/history ─────────────────────────────────────────────────
  Returns filtered GPS breadcrumbs across all time.
  Filter: keep a point only if BOTH conditions met vs. previous kept point:
    • ≥60 seconds elapsed
    • ≥25 metres moved (haversine distance)
  Exception: incident-flagged points always included regardless of filter.
  SPL:
    search index=pta sourcetype="pta:gps_location" earliest=0 latest=now
    | eval ts=_time
    | fields ts latitude longitude incident incidentNote
    | sort 0 ts
  Server-side haversine filtering applied after query.
  Returns: { points: [{lat, lon, ts, incident, incidentNote}] }

  ── GET /api/pta/gps/route ───────────────────────────────────────────────────
  Returns ALL raw GPS points across all time, unfiltered, for drawing the
  yellow route polyline on the map.
  SPL:
    search index=pta sourcetype="pta:gps_location" earliest=0 latest=now
    | eval ts=_time
    | fields ts latitude longitude
    | sort 0 ts
  Returns: { points: [{lat, lon, ts}] }

  ── POST /api/pta/reset ──────────────────────────────────────────────────────
  Deletes all pta:gps_location events from Splunk permanently.
  SPL: search index=pta sourcetype="pta:gps_location" earliest=0 latest=now
       | delete
  Uses: Splunk management API /services/search/jobs (exec_mode=oneshot)
  Requires: can_delete capability (see Splunk Permission Fix below)
  Returns: { "status": "ok", "message": "GPS tracking history cleared" }

  ── GET /pta ─────────────────────────────────────────────────────────────────
  Full Google Maps HTML page served from FastAPI.
  On load:
    1. loadRoute()   — fetches /api/pta/gps/route, draws yellow polyline
    2. loadHistory() — fetches /api/pta/gps/history, places markers
    3. poll()        — fetches /api/pta/gps, moves live bus marker
  Refresh intervals:
    poll():        every 15 seconds
    loadRoute():   every 60 seconds
    loadHistory(): every 60 seconds
  Map auto-centres on latest history point if no live ping is active.

    Click:  InfoWindow shows ID, time, optional note, STATUS: OPEN
    zIndex: 5 (above route and breadcrumbs)

  Bus marker (live position):
    Source: /api/pta/gps (latest GPS fix)
    Icon:   maps.google.com/mapfiles/ms/icons/bus.png
    Click:  InfoWindow shows vehicleId, lat, lon, accuracy, speed, heading
    zIndex: 10 (always on top)
    Hidden until first valid GPS response.

  RESET button:
    Styled: red border/text, fills red on hover
    Action: confirms → POST /api/pta/reset → clears polyline + all markers
            + hides live bus marker

  Incident ID format:
    Generated client-side from event timestamp: INC-MMDD-HHmmSS
    e.g. INC-0623-143022 = 23 June, 14:30:22 Perth time
    Human-readable, unique to the second, no server ID generation required.

================================================================================
LESSONS LEARNED / GOTCHAS
================================================================================

1. MIXED CONTENT BLOCK (most important)
   AI Studio preview (HTTPS) cannot fetch HTTP endpoints.
   Browser silently drops the request — no error shown to user,
   Packets Sent counter stays at 0, nothing reaches the server.
   Fix: HTTPS everywhere. Use `tailscale cert` for a valid cert.

2. RELATIVE URL GOES TO GOOGLE'S CLOUD
   fetch('/api/telemetry') in the AI Studio app resolves to
   https://aistudio.google.com/api/telemetry — Google's servers,
   which cannot reach Tailscale-only addresses.
   Fix: Always use the full absolute URL.

3. SPLUNK HTML PANEL CSP (Splunk 10.x)
   SimpleXML encoded="0" HTML panels do not execute inline JavaScript
   in Splunk 10.x due to Content Security Policy restrictions.
   Fix: Serve the map from FastAPI and embed via encoded="1" iframe.

4. SPLUNK WEB PORT IS 8008 (not 8000)
   This installation runs Splunk Web on port 8008.
   Always confirm with: sudo ss -tlnp | grep splunk

5. ANDROID BLOCKS PING (ICMP)
   `ping` in Termux/Android always fails for non-root — not a network issue.
   Use curl to test connectivity instead.

6. WRONG HEC TOKEN
   The existing .env had SPLUNK_HEC_TOKEN pointing to the meraki-switch
   HEC input. A dedicated pta_gps_feed token was created and stored as
   PTA_HEC_TOKEN in .env to keep the data sources clean.

7. DASHBOARD SHARING
   Dashboards created via REST API default to user-level sharing (private).
   Must explicitly set sharing=app via the ACL endpoint after creation.

8. head 1 WITHOUT sort
   `stats ... BY vehicleId | head 1` returns an arbitrary row.
   Always add `| sort -ts` before `| head 1` to get the most recent vehicle.

9. JSON FIELD EXTRACTION
   Splunk auto-extracts JSON fields from _raw for this sourcetype.
   Adding `| spath` on top causes duplicate multi-value fields.
   Do not use spath — the fields are already available.

10. TAILSCALE IP vs HOSTNAME
    The Tailscale cert is issued for the hostname (brannigan.taila277ca.ts.net),
    not the IP (100.81.70.37). Using the IP with HTTPS causes cert mismatch.
    Always use the full hostname in URLs.

11. SPLUNK _time IS A FORMATTED STRING IN EXPORT API
    When querying via /services/search/jobs/export with output_mode=json,
    the _time field comes back as a formatted string e.g.
    "2026-06-22 14:19:42.000 AWST" — NOT an epoch number.
    float("2026-06-22...") throws ValueError, silently caught, returns nothing.
    Fix: add `| eval ts=_time` in the SPL BEFORE `| fields ...`
    In eval context, _time IS the epoch float. Use `ts` as the field name.

12. SPLUNK can_delete NOT IN DEFAULT ADMIN ROLE
    The `| delete` search command requires the can_delete capability.
    This is NOT granted to the admin role by default in Splunk — by design.
    The REST API returns HTTP 200 with a FATAL message in the JSON body,
    NOT a non-2xx status code. resp.ok is True even on failure.
    Fix: Create /opt/splunk/etc/system/local/authorize.conf with:
      [role_admin]
      importRoles = power;user;can_delete
    Then restart Splunk. The role_can_delete role has delete_by_keyword which
    grants the | delete command. Restart required for conf changes to apply.

13. GPS ENDPOINT TIME WINDOW
    Originally the /api/pta/gps endpoint used earliest_time=-1h.
    If the tracker has been idle >1 hour, it returns found:false even though
    Splunk has valid historical data. The map shows "No GPS pings yet" and
    stays at default Perth CBD centre, hiding all history markers off-screen.
    Fix: Set earliest_time=0 (all time). The SPL `| head 1` ensures only
    the single most recent result is returned regardless of data volume.

14. INCIDENT FLAG — SINGLE TELEMETRY PATH
    Sending incidents as a separate API call or sourcetype adds complexity
    with no benefit. The cleanest design: the phone app adds incident=true
    and incidentNote fields to the standard GPS telemetry payload.
    The server proxy forwards unchanged. Splunk stores everything in
    pta:gps_location. History endpoint surfaces the incident field.
    Incident ID is derived client-side from the GPS timestamp — no
    server-side ID generation or separate event creation required.
